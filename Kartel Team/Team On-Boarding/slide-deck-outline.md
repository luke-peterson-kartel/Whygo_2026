## Design Notes

### Style Guidelines

**Typography**:
- Titles: Bold, 48pt
- Body: Clean sans-serif, 24-32pt
- Minimal text per slide (max 5 bullet points)

**Colors**:
- Primary: Kartel brand colors
- Status indicators: Green (#22C55E), Yellow (#EAB308), Red (#EF4444)
- Background: Clean white or subtle gradient

**Icons**:
- Simple, modern line icons
- Consistent style throughout
- High contrast for visibility

**Images**:
- Actual screenshots of the dashboard (slide 4-5)
- Clean, uncluttered terminal windows
- Highlight key elements with callout boxes

### Animation (Optional)

**Slide 4 - How It Works**:
- Animate the 3-step flow left-to-right
- Each step appears after previous (1 second delay)

**Slide 5 - Status Indicators**:
- Fade in each status tier (top to bottom)
- Progress bars animate to fill level

**Slide 7 - What's Expected**:
- Checklist items appear one-by-one with checkmark animation

## Delivery Tips

1. **Practice the demo** - Run through terminal commands before meeting
2. **Have backup terminal window** - In case something fails
3. **Screen share in HD** - Terminal text needs to be readable
4. **Slow down on demo** - Give people time to see what you're doing
5. **Pause for questions** - After demo, before expectations
6. **End with energy** - This is exciting, not a chore!
